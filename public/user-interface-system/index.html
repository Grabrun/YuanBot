    用户界面

    🌸 缘·Bot 用户界面系统详细设计文档 (v1.5)

    版本历史

    版本 日期 修改内容 v1.0 2026-05-22 初始详细设计，基于总体架构 v1.5 v1.1 2026-05-22 补充聊天记录认证鉴权设计，完善安全架构

    1. 系统定位与目标

    用户界面系统是 缘·Bot 的“面孔与双手”，负责为用户提供自然、高效、多模态的交互入口。它包含：

    · TUI 聊天界面：面向终端爱好者和服务器管理员的轻量、原生终端交互体验。 · WebUI 聊天界面：面向日常用户的现代化、功能丰富、支持多媒体的聊天界面。 · WebUI 管理界面：面向所有用户的图形化系统管理、配置、记忆浏览与插件管理面板。

    核心目标：

    · 全功能覆盖：聊天、历史、配置、记忆管理、插件市场均可在界面中完成。 · 统一体验：TUI 与 WebUI 共享相同的后端能力，仅表现形式不同。 · 开箱即用：无额外配置，启动即提供 WebUI；一行命令进入 TUI。 · 实时反馈：流式对话、在线状态、主动消息即时推送。 · 安全可控：聊天记录、管理接口均需认证鉴权，支持多用户隔离。 · 可访问性：支持键盘导航、终端色彩主题、响应式 Web 设计。

    1. 认证与鉴权体系

    3.1 设计原则

    · 所有聊天数据访问均需认证：无论是 WebUI 还是 TUI，查看历史消息、发起新对话都必须验证用户身份。 · 用户数据隔离：每个用户只能访问自己的会话、消息和记忆。 · 管理接口独立鉴权：系统管理功能需管理员权限，与普通聊天用户分离。 · Token 生命周期管理：支持短期访问令牌和长期 API Key 两种模式。

    3.2 用户身份模型

    @dataclass
class User:
    user_id: str          # 全局唯一 ID
    username: str         # 登录用户名
    display_name: str     # 显示名称
    role: str             # "admin" | "user"
    created_at: datetime
    last_login: datetime

@dataclass
class AuthToken:
    token: str            # JWT 或 Session Token
    user_id: str
    expires_at: datetime
    token_type: str       # "session" | "api_key"

    3.3 认证方式

    认证方式 适用场景 说明 用户名/密码 WebUI 登录 默认方式，首次启动创建管理员账号 API Key TUI、CLI、外部集成 长期令牌，在管理界面生成，可随时吊销 OAuth 2.0 未来扩展 支持 GitHub、Google 等第三方登录 本地信任 本地 TUI 检测到本地回环地址访问且无密码配置时，自动信任

    3.4 鉴权中间件

    所有 UI API 请求经过两层中间件：

    第一层：认证中间件

    · 从请求头 Authorization: Bearer 或 Cookie yuanbot_token 中提取令牌。 · 验证令牌有效性（签名、过期时间）。 · 将解析出的 user_id 和 role 注入请求上下文。

    第二层：鉴权守卫

    · 检查请求路径所需的权限级别： · /api/chat/：需要已认证用户。 · /api/admin/：需要 role: admin。 · /api/system/**：需要 role: admin。 · 检查资源归属：查询聊天记录、记忆等时，自动过滤当前 user_id，防止越权。

    3.5 聊天记录访问控制

    接口 认证要求 鉴权规则 POST /api/chat 必须 使用 token 中的 user_id 创建/追加消息 GET /api/conversations 必须 仅返回当前用户的会话列表 GET /api/conversations/{id} 必须 验证会话归属当前用户 GET /api/conversations/{id}/messages 必须 验证会话归属当前用户 DELETE /api/conversations/{id} 必须 验证会话归属当前用户 WebSocket /ws/chat 必须 连接时验证 token，仅推送该用户的消息流

    功能 说明 流式对话 实时显示 AI 回复，逐字渲染，支持 Markdown 基础格式（粗体、斜体、代码块）。 多会话管理 Ctrl+N 新建会话，Ctrl+Tab 切换；每个会话保留独立上下文。 斜杠命令 /help 帮助，/memory 查看/搜索记忆，/persona 查看/切换人设，/plugin list 列出插件，/provider 查看 AI 提供商状态，/clear 清屏，/quit 退出。 认证命令 /login 输入用户名密码，/logout 退出登录，/token 显示当前 API Key。 表情与颜色 终端内渲染 Unicode 表情；柔和配色主题，可根据终端背景色自适应。 消息历史 从服务端拉取历史，支持 /history search 搜索。 消息通知 当处于其他会话或后台时，新消息通过终端响铃 (\a) 或标题变化提醒。

# 或独立启动，自动拉起核心

    1. WebUI 管理界面

    管理界面是 WebUI 的一个独立路由 (/admin)，仅管理员角色可访问。

    6.1 访问控制

    · 需登录为 role: admin 的用户。 · 普通用户访问 /admin 重定向回聊天界面。 · 管理 API 端点 (/api/admin/**) 在服务端进行二次鉴权。

    6.2 功能模块

    6.2.1 仪表盘

    · 系统状态：CPU/内存/磁盘使用率，数据库连接状态。 · AI 提供商：活跃提供商、Token 用量（今日/本月）、费用估算。 · 消息统计：今日消息数、活跃用户数、主动消息发送数。 · 记忆统计：总事实记忆数、情景记忆数、图谱节点数。

    6.2.2 用户管理

    · 用户列表：查看所有注册用户。 · 创建用户：管理员创建新用户账号。 · 角色管理：修改用户角色（user/admin）。 · API Key 管理：为用户生成/吊销 API Key。 · 强制下线：使某用户的 token 失效。

    6.2.3 记忆浏览器

    · 事实记忆列表：表格展示当前用户的所有事实，支持分类筛选、搜索、编辑、删除。 · 情景记忆时间线：以时间轴展示历史对话摘要，点击查看详情，可手动淘汰。 · 记忆图谱可视化：使用 ECharts/D3.js 绘制实体-关系图，可拖拽、缩放。

    6.2.4 配置编辑器

    · 全局配置：在线编辑 bot.yaml，实时语法检查，保存后热重载。 · Provider 配置：可视化添加/编辑/删除 AI 提供商。 · Channel 配置：管理消息通道，开关通道。 · 插件管理：查看已安装 Skills/Tools，启用/禁用，安装社区插件。

    6.2.5 人格商店

    · 浏览社区分享的人设包，查看详情、评分、评论。 · 一键安装到本地，自动激活或手动切换。

    6.2.6 日志查看

    · 实时日志流（WebSocket），支持级别过滤、关键字搜索。 · 历史日志文件下载。

    6.2.7 系统维护

    · 备份与恢复：一键备份数据库和配置，从备份文件恢复。 · 数据导出：导出记忆、对话历史为 JSON/CSV。 · 隐私管理：清除所有记忆、重置系统。 · 更新检查：检查核心和扩展更新，在线升级。

    1. UI API 层设计

    UI API 层是 TUI 和 WebUI 的共同后端，所有端点（除登录外）均需认证。

    7.1 认证相关端点

    方法/路径 认证 说明 POST /api/auth/login 否 用户名/密码登录，返回 JWT token POST /api/auth/api-key 否 API Key 验证，返回 Session token POST /api/auth/logout 是 注销当前 token POST /api/auth/refresh 是 刷新即将过期的 token GET /api/auth/me 是 获取当前用户信息

    7.2 聊天相关端点

    方法/路径 认证 鉴权 说明 POST /api/chat 是 用户 发送消息，使用当前 user_id GET /api/conversations 是 用户 获取当前用户的会话列表 POST /api/conversations 是 用户 新建会话 GET /api/conversations/{id} 是 用户 获取会话详情（校验归属） DELETE /api/conversations/{id} 是 用户 删除会话（校验归属） GET /api/conversations/{id}/messages 是 用户 获取历史消息（校验归属）

    7.3 管理相关端点

    方法/路径 认证 鉴权 说明 GET /api/admin/users 是 管理员 用户列表 POST /api/admin/users 是 管理员 创建用户 DELETE /api/admin/users/{id} 是 管理员 删除用户 POST /api/admin/users/{id}/api-key 是 管理员 生成 API Key DELETE /api/admin/users/{id}/api-key 是 管理员 吊销 API Key GET /api/admin/metrics 是 管理员 系统指标 POST /api/admin/backup 是 管理员 触发备份 POST /api/admin/restore 是 管理员 从备份恢复

    7.4 WebSocket 端点

    路径 认证 说明 /ws/chat?token= 是 流式聊天，连接时验证 token，仅推送该用户的消息 /ws/logs?token= 是（管理员） 实时日志流

    WebSocket 认证流程：

    1. 客户端连接时在 URL 参数中携带 token。
    2. 服务端在握手阶段验证 token。
    3. 验证失败 → 拒绝连接（返回 401）。
    4. 验证成功 → 建立连接，将 user_id 绑定到该连接。
    1. 安全机制总结

    安全层 机制 传输安全 全站 HTTPS（TLS 1.2+），WSS 加密 WebSocket 认证 JWT Token + Session Cookie，支持 API Key 鉴权 基于角色的访问控制（RBAC），资源归属校验 聊天记录隔离 所有查询自动过滤 user_id，确保用户只能访问自己的数据 管理接口保护 仅 admin 角色可访问管理 API 和页面 Token 安全 短期 JWT（默认 24h）+ 刷新机制；API Key 可随时吊销 输入校验 所有用户输入在服务端二次校验，防注入 CORS 配置 可配置允许的源，防止跨站请求伪造
