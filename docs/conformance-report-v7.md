## V7 具体修复内容

### 1. Ruff Lint 清零 (20 errors → 0)

| # | 文件 | 错误类型 | 修复方式 |

| 3 | `sqlite_store.py:146` | E501 行过长 (105>100) | 拆分 SQL trigger INSERT 语句 |
| 4 | `sqlite_store.py:149` | E501 行过长 (105>100) | 拆分 SQL trigger INSERT 语句 |
| 5 | `sqlite_store.py:695` | E501 行过长 (103>100) | 拆分 INSERT OR REPLACE SQL 语句 |
| 6 | `sqlite_store.py:741` | UP017 `timezone.utc` | 改用 `datetime.UTC` |
| 7 | `sqlite_store.py:764` | UP017 `timezone.utc` | 改用 `datetime.UTC` |
| 8-20 | `tools/proto/*` | I001/F401/N802/E501/UP009 | 通过 ruff exclude 排除自动生成文件 |

### 2. gRPC 客户端完善

**SandboxClient.list_tools()** — 完整实现：
- 检查 gRPC/proto 可用性 + 连接状态
- 通过 `ToolExecutorServiceStub.ListTools` RPC 获取工具列表
- 将 `pb2.ToolInfo` 转换为 `ToolInfoData` 数据类
- 异常处理 + 降级返回空列表

**SandboxClient.health_check()** — 完整实现：
- 检查 gRPC/proto 可用性 + 连接状态
- 通过 `ToolExecutorServiceStub.HealthCheck` RPC 获取健康状态
- 返回 `{healthy, version, active_executions}` 完整信息
- 异常处理 + 降级返回 `{"healthy": False}`

### 3. pyproject.toml 配置优化

```toml
[tool.ruff]
target-version = "py312"
line-length = 100
exclude = ["src/yuanbot/tools/proto/*"]
```

排除自动生成的 protobuf/gRPC 文件，避免干扰 lint 检查。
