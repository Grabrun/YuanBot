1. 系统定位与目标

用户界面系统是 缘·Bot 的“面孔与双手”，负责为用户提供自然、高效、多模态的交互入口。它包含：

· TUI 聊天界面：面向终端爱好者和服务器管理员的轻量、原生终端交互体验。
· WebUI 聊天界面：面向日常用户的现代化、功能丰富、支持多媒体的聊天界面。
· WebUI 管理界面：面向所有用户的图形化系统管理、配置、记忆浏览与插件管理面板。

核心目标：

· 全功能覆盖：聊天、历史、配置、记忆管理、插件市场均可在界面中完成。
· 统一体验：TUI 与 WebUI 共享相同的后端能力，仅表现形式不同。
· 开箱即用：无额外配置，启动即提供 WebUI；一行命令进入 TUI。
· 实时反馈：流式对话、在线状态、主动消息即时推送。
· 安全可控：聊天记录、管理接口均需认证鉴权，支持多用户隔离。
· 可访问性：支持键盘导航、终端色彩主题、响应式 Web 设计。

3. 认证与鉴权体系

3.1 设计原则

· 所有聊天数据访问均需认证：无论是 WebUI 还是 TUI，查看历史消息、发起新对话都必须验证用户身份。
· 用户数据隔离：每个用户只能访问自己的会话、消息和记忆。
· 管理接口独立鉴权：系统管理功能需管理员权限，与普通聊天用户分离。
· Token 生命周期管理：支持短期访问令牌和长期 API Key 两种模式。

3.2 用户身份模型

```python
@dataclass
class User:
    user_id: str          # 全局唯一 ID
    username: str         # 登录用户名
    display_name: str     # 显示名称
    role: str             # "admin" | "user"
    created_at: datetime
    last_login: datetime

@dataclass
class AuthToken:
    token: str            # JWT 或 Session Token
    user_id: str
    expires_at: datetime
    token_type: str       # "session" | "api_key"
```

3.3 认证方式

认证方式 适用场景 说明
用户名/密码 WebUI 登录 默认方式，首次启动创建管理员账号
API Key TUI、CLI、外部集成 长期令牌，在管理界面生成，可随时吊销
OAuth 2.0 未来扩展 支持 GitHub、Google 等第三方登录
本地信任 本地 TUI 检测到本地回环地址访问且无密码配置时，自动信任

3.4 鉴权中间件

所有 UI API 请求经过两层中间件：

第一层：认证中间件

· 从请求头 Authorization: Bearer <token> 或 Cookie yuanbot_token 中提取令牌。
· 验证令牌有效性（签名、过期时间）。
· 将解析出的 user_id 和 role 注入请求上下文。

第二层：鉴权守卫

· 检查请求路径所需的权限级别：
  · /api/chat/**：需要已认证用户。
  · /api/admin/**：需要 role: admin。
  · /api/system/**：需要 role: admin。
· 检查资源归属：查询聊天记录、记忆等时，自动过滤当前 user_id，防止越权。

3.5 聊天记录访问控制

接口 认证要求 鉴权规则
POST /api/chat 必须 使用 token 中的 user_id 创建/追加消息
GET /api/conversations 必须 仅返回当前用户的会话列表
GET /api/conversations/{id} 必须 验证会话归属当前用户
GET /api/conversations/{id}/messages 必须 验证会话归属当前用户
DELETE /api/conversations/{id} 必须 验证会话归属当前用户
WebSocket /ws/chat 必须 连接时验证 token，仅推送该用户的消息流

3.6 WebUI 登录流程

```
用户访问 WebUI
    │
    ▼
检测是否已登录（Cookie 中是否有有效 token）
    │
    ├── 已登录 → 直接进入聊天界面
    │
    └── 未登录 → 跳转登录页面
                    │
                    ├── 输入用户名/密码
                    │       │
                    │       ▼
                    │   POST /api/auth/login
                    │       │
                    │       ├── 验证成功 → 返回 token + 设置 Cookie
                    │       │              → 跳转聊天界面
                    │       │
                    │       └── 验证失败 → 显示错误提示
                    │
                    └── 使用 API Key 登录
                            │
                            ▼
                        在登录页输入 API Key
                            │
                            ▼
                        验证成功 → 存储 token → 跳转聊天界面
```

3.7 TUI 登录流程

```
启动 yuanbot-cli tui
    │
    ▼
检测是否配置了 API Key（环境变量/配置文件）
    │
    ├── 已配置 → 直接连接
    │
    └── 未配置 → 检测本地连接
                    │
                    ├── 本地回环 + 无管理员密码 → 自动信任模式
                    │
                    └── 远程或需密码 → 交互式输入用户名/密码
```

TUI 配置方式：

· 环境变量：YUANBOT_API_KEY=xxx
· 配置文件：~/.yuanbot/config.yaml 中的 api_key 字段
· 交互输入：启动时提示输入

6. WebUI 管理界面

管理界面是 WebUI 的一个独立路由 (/admin)，仅管理员角色可访问。

6.1 访问控制

· 需登录为 role: admin 的用户。
· 普通用户访问 /admin 重定向回聊天界面。
· 管理 API 端点 (/api/admin/**) 在服务端进行二次鉴权。

6.2 功能模块

6.2.1 仪表盘

· 系统状态：CPU/内存/磁盘使用率，数据库连接状态。
· AI 提供商：活跃提供商、Token 用量（今日/本月）、费用估算。
· 消息统计：今日消息数、活跃用户数、主动消息发送数。
· 记忆统计：总事实记忆数、情景记忆数、图谱节点数。

6.2.2 用户管理

· 用户列表：查看所有注册用户。
· 创建用户：管理员创建新用户账号。
· 角色管理：修改用户角色（user/admin）。
· API Key 管理：为用户生成/吊销 API Key。
· 强制下线：使某用户的 token 失效。

6.2.3 记忆浏览器

· 事实记忆列表：表格展示当前用户的所有事实，支持分类筛选、搜索、编辑、删除。
· 情景记忆时间线：以时间轴展示历史对话摘要，点击查看详情，可手动淘汰。
· 记忆图谱可视化：使用 ECharts/D3.js 绘制实体-关系图，可拖拽、缩放。

6.2.4 配置编辑器

· 全局配置：在线编辑 bot.yaml，实时语法检查，保存后热重载。
· Provider 配置：可视化添加/编辑/删除 AI 提供商。
· Channel 配置：管理消息通道，开关通道。
· 插件管理：查看已安装 Skills/Tools，启用/禁用，安装社区插件。

6.2.5 人格商店

· 浏览社区分享的人设包，查看详情、评分、评论。
· 一键安装到本地，自动激活或手动切换。

6.2.6 日志查看

· 实时日志流（WebSocket），支持级别过滤、关键字搜索。
· 历史日志文件下载。

6.2.7 系统维护

· 备份与恢复：一键备份数据库和配置，从备份文件恢复。
· 数据导出：导出记忆、对话历史为 JSON/CSV。
· 隐私管理：清除所有记忆、重置系统。
· 更新检查：检查核心和扩展更新，在线升级。

7. UI API 层设计

UI API 层是 TUI 和 WebUI 的共同后端，所有端点（除登录外）均需认证。

7.1 认证相关端点

方法/路径 认证 说明
POST /api/auth/login 否 用户名/密码登录，返回 JWT token
POST /api/auth/api-key 否 API Key 验证，返回 Session token
POST /api/auth/logout 是 注销当前 token
POST /api/auth/refresh 是 刷新即将过期的 token
GET /api/auth/me 是 获取当前用户信息

7.2 聊天相关端点

方法/路径 认证 鉴权 说明
POST /api/chat 是 用户 发送消息，使用当前 user_id
GET /api/conversations 是 用户 获取当前用户的会话列表
POST /api/conversations 是 用户 新建会话
GET /api/conversations/{id} 是 用户 获取会话详情（校验归属）
DELETE /api/conversations/{id} 是 用户 删除会话（校验归属）
GET /api/conversations/{id}/messages 是 用户 获取历史消息（校验归属）

7.3 管理相关端点

方法/路径 认证 鉴权 说明
GET /api/admin/users 是 管理员 用户列表
POST /api/admin/users 是 管理员 创建用户
DELETE /api/admin/users/{id} 是 管理员 删除用户
POST /api/admin/users/{id}/api-key 是 管理员 生成 API Key
DELETE /api/admin/users/{id}/api-key 是 管理员 吊销 API Key
GET /api/admin/metrics 是 管理员 系统指标
POST /api/admin/backup 是 管理员 触发备份
POST /api/admin/restore 是 管理员 从备份恢复

7.4 WebSocket 端点

路径 认证 说明
/ws/chat?token=<jwt> 是 流式聊天，连接时验证 token，仅推送该用户的消息
/ws/logs?token=<jwt> 是（管理员） 实时日志流

WebSocket 认证流程：

1. 客户端连接时在 URL 参数中携带 token。
2. 服务端在握手阶段验证 token。
3. 验证失败 → 拒绝连接（返回 401）。
4. 验证成功 → 建立连接，将 user_id 绑定到该连接。

8. 安全机制总结

安全层 机制
传输安全 全站 HTTPS（TLS 1.2+），WSS 加密 WebSocket
认证 JWT Token + Session Cookie，支持 API Key
鉴权 基于角色的访问控制（RBAC），资源归属校验
聊天记录隔离 所有查询自动过滤 user_id，确保用户只能访问自己的数据
管理接口保护 仅 admin 角色可访问管理 API 和页面
Token 安全 短期 JWT（默认 24h）+ 刷新机制；API Key 可随时吊销
输入校验 所有用户输入在服务端二次校验，防注入
CORS 配置 可配置允许的源，防止跨站请求伪造

10. 可访问性与国际化

· 键盘导航：TUI 全键盘操作；WebUI 支持 Tab 键焦点移动、Esc 关闭弹窗。
· 屏幕阅读器：WebUI 的 ARIA 标签、语义化 HTML。
· 多语言：界面文本支持中/英/日，根据浏览器语言自动切换，可在设置中手动选择。

---

11. 部署与集成

· TUI：随 yuanbot-cli 发布，通过 pip install yuanbot-cli 获得。
· WebUI：前端资源编译后作为 Python 包数据文件内嵌在 yuanbot-core 中，核心启动时自动提供静态文件服务，无需单独部署。
· 独立部署：也可以将 WebUI 静态文件提取，由 Nginx 代理，核心仅提供 API。
